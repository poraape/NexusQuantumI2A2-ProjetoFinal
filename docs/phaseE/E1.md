# E1 — Versionamento e Catálogo de Capacidades

## Objetivo
Implementar versionamento de agentes e catálogo de ferramentas com depreciação gradual documentada.

## Entradas
- Registro `catalog/agents.yaml`
- Migrações `migration/*.md`
- Tests de compatibilidade `tests/compatibility.spec.mjs`

## Passos
1. Registrar agentes com versionamento semântico (`catalog/agents.yaml`).
2. Criar migrações incrementais descrevendo mudanças (`migration/`).
3. Executar testes de compatibilidade garantindo APIs anteriores funcionais.
4. Publicar changelog em `contracts/CHANGELOG.md`.

## Saídas
- `catalog/agents.yaml`
- `migration/001-agent-versioning.md`
- `tests/compatibility.spec.mjs`

## Critérios de Aceitação
- Compatibilidade reversa confirmada.
- Depreciações comunicadas com antecedência ≥2 ciclos.
- Ferramentas catalogadas por owner e SLA.
