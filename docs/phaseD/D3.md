# D3 — Governança de Modelos

## Objetivo
Catalogar modelos, datasets e critérios de aceitação para habilitar rollback automatizado.

## Entradas
- Registro `catalog/agents.yaml`
- Template `models/catalog.yaml`
- Testes `tests/model_regression.spec.mjs`

## Passos
1. Preencher `models/catalog.yaml` com metadados (versão, fornecedor, custo).
2. Adicionar critérios de aceitação e métricas em `reports/weekly/template.md`.
3. Automatizar regressão via `tests/model_regression.spec.mjs` comparando outputs com baseline.
4. Integrar ao pipeline CI com gate de aprovação humana.

## Saídas
- `models/catalog.yaml`
- `tests/model_regression.spec.mjs`
- Atualização de `contracts/CHANGELOG.md` quando modelos impactam contratos.

## Critérios de Aceitação
- Cada modelo possui owner, SLO e métricas históricas.
- Rollback disponível em menos de 5 minutos.
- Logs de decisão armazenados em `audits/model-decisions.ndjson`.
